Jaringan Komputer & Internet

Materi ringkas dan terstruktur tentang konsep jaringan, model, perangkat, protokol, keamanan, serta latihan praktis.

Pengantar

Jaringan komputer adalah kumpulan dua atau lebih perangkat yang saling terhubung untuk berbagi sumber daya dan informasi. Internet adalah jaringan global yang menghubungkan jutaan jaringan dan perangkat di seluruh dunia.

Model Referensi: OSI & TCP/IP

OSI (7 lapis)

  1. Physical ‚Äî fisik transmisi bit
  2. Data Link ‚Äî framing, MAC address
  3. Network ‚Äî pengalamatan & routing (IP)
  4. Transport ‚Äî end-to-end (TCP/UDP)
  5. Session ‚Äî pengelolaan sesi
  6. Presentation ‚Äî format data, enkripsi
  7. Application ‚Äî layanan user (HTTP, SMTP)

TCP/IP (4 lapis)

Application, Transport, Internet, Link ‚Äî model yang lebih praktis dan umum digunakan pada internet.

Perangkat Jaringan

Router

Mengarahkan paket antar jaringan (menggunakan tabel routing).

Switch

Menyambungkan perangkat dalam jaringan lokal, bekerja pada layer data-link (MAC).

Modem & Access Point

Modem menghubungkan ke ISP; Access Point menyediakan koneksi nirkabel (Wi‚ÄëFi).

Firewall

Memfilter lalu lintas berdasarkan aturan keamanan.

IP Address & Subnetting (Dasar)

IP address mengidentifikasi perangkat di jaringan. Versi umum: IPv4 (xxx.xxx.xxx.xxx) dan IPv6.

Subnetting singkat

Subnet mask memisah bagian network dan host. Contoh: 192.168.1.0/24 berarti 24 bit untuk network (256 alamat total, 254 host usable).

Contoh: 192.168.1.10/24 => Network: 192.168.1.0, Broadcast: 192.168.1.255

DNS & DHCP

DNS menerjemahkan nama domain (contoh.com) ke alamat IP. DHCP memberikan alamat IP otomatis ke perangkat di jaringan.

Routing vs Switching

Switching menghubungkan perangkat di jaringan yang sama (LAN); routing menghubungkan antar jaringan (menggunakan IP) dan memilih jalur terbaik.

Protokol Penting

  • HTTP / HTTPS: protokol web; HTTPS terenkripsi oleh TLS/SSL.
  • TCP: koneksi andal, urutan & retransmisi.
  • UDP: tanpa koneksi, cepat (streaming, game).
  • ICMP: pesan kontrol (ping).
  • SSH: remote shell aman.

Jaringan Nirkabel (Wi‚ÄëFi)

Standar umum: 802.11a/b/g/n/ac/ax. Perhatikan keamanan: gunakan WPA2/WPA3, jangan biarkan SSID tanpa pengamanan.

Keamanan Jaringan

  • Gunakan firewall dan kebijakan akses.
  • Segmentasi jaringan (VLAN) untuk membatasi akses antar grup.
  • Enkripsi lalu lintas sensitif (TLS, VPN).
  • Patch perangkat dan gunakan monitoring untuk deteksi intrusi.

Troubleshooting Dasar

  1. Periksa koneksi fisik (kabel, lampu indikator).
  2. Gunakan ipconfig / ifconfig untuk cek alamat IP.
  3. Gunakan ping untuk verifikasi konektivitas.
  4. Traceroute untuk melihat jalur paket.
  5. Periksa DNS jika nama domain tidak terresolve.

Latihan & Proyek Mini

  • Konfigurasi jaringan LAN sederhana: hubungkan 4 PC dan beri IP statis/ DHCP.
  • Setup Wi‚ÄëFi dengan SSID terpisah untuk tamu dan karyawan (VLAN sederhana).
  • Konfigurasi port forwarding pada router untuk web server lokal.
  • Uji dan dokumentasikan hasil ping, traceroute, dan nslookup.

Catatan: lakukan percobaan pada lingkungan yang aman (lab) untuk menghindari gangguan jaringan publik.
