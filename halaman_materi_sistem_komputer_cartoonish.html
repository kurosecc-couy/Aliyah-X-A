Sistem Komputer

Ringkasan konsep inti sistem komputer: arsitektur, komponen, operasi sistem, penyimpanan, I/O, virtualisasi, dan perawatan.

Definisi Sistem Komputer

Sistem komputer adalah gabungan perangkat keras (hardware), perangkat lunak (software), serta pengguna yang bekerja bersama untuk memproses, menyimpan, dan mengirim informasi. Sistem ini mencakup CPU, memori, perangkat penyimpanan, subsistem I/O, dan perangkat lunak pendukung seperti sistem operasi.

Arsitektur Komputer (Garis Besar)

Model Von Neumann

Model klasik di mana program dan data disimpan dalam memori yang sama. Komponen utama: CPU (ALU + CU), memori utama, dan perangkat I/O.

Perbedaan dengan Harvard Architecture

Harvard memisahkan memori program dan data ‚Äî umum pada sistem embedded untuk kecepatan dan keamanan.

CPU & Proses Eksekusi

CPU menjalankan instruksi program melalui siklus fetch-decode-execute. Kecepatan CPU dipengaruhi oleh frekuensi clock, jumlah core, dan arsitektur pipeline.

Istilah penting

  • Clock speed: frekuensi operasi (GHz)
  • Cores & Threads: unit pemrosesan paralel
  • Cache (L1/L2/L3): memori kecil dan cepat dekat CPU

Memori & Penyimpanan

Jenis memori

  • RAM: memori volatile untuk data aktif
  • ROM/BIOS: instruksi startup perangkat
  • Penyimpanan sekunder: HDD, SSD ‚Äî untuk data persisten

Hierarchy & latensi

Semakin dekat ke CPU (cache) semakin cepat ‚Äî trade-off antara kapasitas dan latensi.

Input / Output (I/O)

Perangkat I/O menghubungkan sistem komputer ke dunia luar: keyboard, mouse, monitor, printer, sensor, dan perangkat jaringan. Metode transfer data termasuk polling, interrupt, dan DMA (Direct Memory Access).

Bus & Interkoneksi

Bus mengangkut alamat, data, dan sinyal kontrol antar komponen. Contoh modern: PCIe untuk peripheral, SATA/ NVMe untuk penyimpanan.

Sistem Operasi (Peran Utama)

Sistem operasi mengelola sumber daya: scheduling CPU, manajemen memori, sistem file, dan antarmuka pengguna. Contoh: Linux, Windows, macOS.

Fungsi inti

  • Manajemen proses & multitasking
  • Manajemen memori (virtual memory, paging)
  • Manajemen file & disk
  • Device drivers untuk perangkat keras

Virtualisasi & Container

Virtualisasi memungkinkan menjalankan beberapa sistem operasi di atas satu mesin fisik melalui hypervisor (VMware, KVM). Containers (Docker) menyediakan isolasi proses lebih ringan tanpa kernel terpisah.

Sistem Embedded & IoT

Sistem embedded adalah komputer khusus yang tertanam dalam perangkat (mis. mikrokontroler pada mesin cuci, sensor pintar). Biasanya resource-constrained dan mengadopsi arsitektur efisien.

Perawatan & Troubleshooting

  • Update firmware & software secara berkala
  • Periksa kesehatan penyimpanan (S.M.A.R.T.)
  • Gunakan monitoring resource (CPU, memory, disk I/O)
  • Langkah troubleshooting: isolate, reproduce, fix, verify

Pengukuran Performa

Metode benchmarking mengukur throughput, latency, IOPS (untuk disk), dan penggunaan CPU/memori. Penting untuk membandingkan konfigurasi dan mengidentifikasi bottleneck.

Keamanan Sistem

  • Amankan firmware & boot chain (secure boot)
  • Patch kerentanan OS dan driver
  • Gunakan enkripsi untuk data sensitif
  • Batasi akses fisik dan jaringan ke perangkat kritis

Latihan & Proyek Mini

  • Rakit PC sederhana dari komponen (CPU, motherboard, RAM, penyimpanan) dan catat langkahnya.
  • Pasang Linux di VM, amati penggunaan resource saat menjalankan aplikasi.
  • Desain topologi untuk server kecil: web server + database + load balancer.
  • Uji performa disk dengan benchmark sederhana dan dokumentasikan hasil.

Selalu lakukan backup sebelum eksperimen yang beresiko data.
